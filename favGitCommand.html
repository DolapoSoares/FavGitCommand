MY FAVOURITE GIT-COMMAND

What is my Favourite git command?

My favourite git command is the git rm command



What does it do?

The git rm command can be used to remove individual files or a collection of files. The primary function of git rm is to remove tracked files from the Git index.



Arguments/flags that can be used with the git rm command

  • git rm -f : The -foption is used to override the safety check that Git makes to ensure that the files in HEAD match the current content in the staging index and working directory.
  • git rm -r : The -r option is shorthand for 'recursive'. When operating in recursive mode git rm will remove a target directory and all the contents of that directory
  • git rm --cached : The cached option specifies that the removal should happen only on the staging index.
  • git rm -q : The quiet option hides the output of the git rm command. The command normally outputs one line for each file removed.
  • git rm -n : The "dry run" option is a safeguard that will execute the git rm command but not actually delete the files. Instead it will output which files it would have removed.
